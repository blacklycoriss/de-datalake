# Работа с Git

## ПРОЦЕСС РАБОТЫ С GIT

1. Скачиваешь изменения из репозитория
2. Меняешь ветку (если работаешь на той, которую нельзя трогать просто так)
3. Кодишь код
4. Коммит
5. Пушишь в репозиторий
6. Делаешь MR (Merge Request - запрос на изменение)
7. После подтверждения изменений удаляешь рабочую ветку
8. Повторяешь заново

Все нужные команды находятся в блоке **ОСНОВНЫЕ КОМАНДЫ**.

Пример:
Я проснулся и улыбнулся, пошел в свой VS Code. Скачиваю изменения из репозитория. Вижу, что работаю в master/dev ветке -> переключаюсь на новую и делаю свою работу там.
После того, как я написал код (есть какое-то решение) - принимаю изменения у себя локально и делаю коммит, после чего отправляю код на сервер.
Захожу в гитхаб, делаю MR и отправляю на проверку. Жду, пока ревьюер проснется и одобрит код. Код одобрен -> иду локально и удаленно удаляю свою ветку, после чего начинаю процесс заново.

## ОСНОВНЫЕ КОМАНДЫ
### Скачивание изменений из репозитория

1. Скачивание и применение на ветку (основная команда)
```bash
git pull
```

**Альтернатива:**
2. Скачивание
```bash
git fetch
```

3. Слияние с кодом
```bash
git merge
```

### Работа с ветками

**Ветка** - ответвление от основной (master/main) линии проекта, в котором происходят изменения.
Пример: если разработка ведется на ветке *dev*, то изменения, внесенные в рамках этой ветки, не видны в рамках ветки *master*.

1. Сменить текущую ветку на другую + создать новую ветку (опция *-b*).
**parent_branch** - ветка, от которой ты унаследовался (предыдущая основная ветка, максимум одна).
**branch_name** - имя новой ветки
```bash
git checkout -b parent_branch/branch_name
```

2. Сменить ветку (без приколов)
Просто меняешь текущую ветку на другую существующую
**branch_name** - в данном случае подразумевается либо *branch_name*, либо *parent_branch* + *branch_name* в формате *parent_branch/branch_name*.
Этой командой можешь переходить сразу к любой ветке.
```bash
git checkout branch_name
```

3. Просмотр веток
```bash
git branch -- просмотр всех локальных веток (на твоем ПК)
git branch -r -- просмотр всех удаленных веток (на GitHub/сервере, где развернут репозиторий)
git branch -a -- просмотр и локальных, и удаленных веток
```

4. Переименование ветки
**new_name** - смотри определение *branch_name*.
```bash
git branch -m new_name -- на текущей ветке
git branch -m old_name new_name -- любая произвольная ветка переименовывается
```

5. Удаление ветки
```bash
git branch -D branch_name -- сначала удаляешь локально
git push origin --delete branch_name -- потом удаляешь на сервере/GitHub
```

6. Добавление файлов в отслеживание
Без этого я не могу сделать коммит.
**Коммит** - фиксация изменений с комментарием, который одной короткой фразой поясняет, что случилось.
```bash
git add . -- добавляю все файлы, которые в текущей директории и в дочерних директориях
```

Кстати говоря, перед выполнением добавления файлов в отслеживание, полезной практикой будет смотреть, какие файлы вообще у тебя есть в отслеживании.
Использую команду:
```bash
git status
```

7. Коммит
Фиксация изменений.
**-m** значит, что коммит с сообщением. Если не ставить, то тебя перекинет в окно, которое попросит оставить текст коммита.
```bash
git commit -m 'Текст моего коммита.'
```

8. Отправка изменений на сервер/GitHub
**serv_name** - как называется репозиторий сервера у тебя на ПК. Как правило, это *origin*.
**branch_name** - ну ты понял.
**-u** - отслеживать изменения этой ветки (не парься, просто ставь всегда).
```bash
git push -u serv_name/branch_name
```
